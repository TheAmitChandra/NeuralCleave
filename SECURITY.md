# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in NeuralCleave, please **do not** open a public GitHub issue.

Instead, report it privately to **ask.amitchandra@gmail.com** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce it
- Affected version(s) — output of `neuralcleave --version`

You should expect an initial response within a few days. Once a fix is available, we'll coordinate on disclosure timing and credit (if you'd like it) in the release notes.

## Supported Versions

Only the latest released version of NeuralCleave receives security fixes.

## Scope

This covers the NeuralCleave gateway, channel adapters, plugin SDK, and desktop app. It does not cover third-party plugins distributed via the NeuralCleave Hub unless the vulnerability is in the Hub/plugin-loading mechanism itself.
